# Architecture

AI OS is a Node.js CLI tool (TypeScript, ESM) that scans a repository, generates GitHub Copilot context files, and manages an MCP server for live context retrieval.

## High-Level Flow

```
npx github:marinvch/ai-os
       │
       ▼
  CLI args parsed (src/cli/args.ts)
       │
       ▼
  Action dispatched (src/cli/dispatch.ts)
       │
       ├─── apply (default / --refresh-existing)
       ├─── dry-run
       ├─── doctor
       ├─── check-freshness
       ├─── check-hygiene
       ├─── compact-memory
       └─── uninstall
```

## Core Modules

### Detectors (`src/detectors/`)

Analyze the repository to build a `ProjectInfo` object:

- **`language.ts`** — detects primary language and secondary languages
- **`framework.ts`** — identifies framework (Next.js, Express, FastAPI, etc.)
- **`patterns.ts`** — finds naming conventions, test framework, linter
- **`graph.ts`** — builds a module dependency graph
- **`freshness.ts`** — scores context staleness vs. last generation snapshot

### Generators (`src/generators/`)

Transform `ProjectInfo` into files on disk:

- **`instructions.ts`** — generates `copilot-instructions.md`
- **`context-docs.ts`** — generates architecture, conventions, stack docs
- **`mcp.ts`** — generates `.vscode/mcp.json` and `.mcp.json`
- **`skills.ts`** — selects and installs skills for the detected stack
- **`agents.ts`** — generates agent files
- **`prompts.ts`** — generates prompt template files
- **`workflows.ts`** — generates CI workflow files
- **`utils.ts`** — shared: `writeIfChanged`, content hashing, dry-run capture, manifests

### MCP Server (`src/mcp-server/`)

Serves the 30 MCP tools over the Model Context Protocol:

- **`index.ts`** — server entry point, routes tool calls
- **`tool-definitions.ts`** — all 30 tool definitions with schema and handler
- **`project-introspection.ts`** — file summary, dependency chain, stack info
- **`memory.ts`** — repo memory read/write
- **`session.ts`** — session state, active plan, checkpoints
- **`freshness-bridge.ts`** — connects freshness detector to MCP tools
- **`search.ts`** — semantic codebase search

### Recommendations (`src/recommendations/`)

Suggests tools, skills, and extensions based on detected stack:

- **`registry.ts`** — curated recommendation registry
- **`index.ts`** — filters registry by detected stack
- **`cli-compat.ts`** — builds skills CLI install commands for different CLI modes

## Content-Hash Gate

Every generated file goes through `writeIfChanged` in `src/generators/utils.ts`:

1. Compute SHA-256 of generated content
2. Compare against previous-run hash stored in `bundle-manifest.json`
3. If unchanged, skip disk write entirely (fast-path)
4. If changed, compare with actual file on disk before writing
5. Record new hash for the next run

This means refreshes are O(changed files), not O(all files).

## MCP Tool Partitioning

When `strictStackFiltering: true` (default):

- `getMcpToolsPartitioned()` separates tools into `activeTools` and `availableButInactive`
- Only `activeTools` are written to `.vscode/mcp.json`
- `tools.json` records both lists for transparency
- Runtime server reads `tools.json` via `getActiveToolsForProject()`

## Bundle

The CLI is distributed as a single-file bundle (`bundle/generate.js`) built with esbuild. The MCP server is bundled separately as `dist/server.js`. Both are generated by `npm run bundle`.
